我准备做一个基于 AI agent 的小说创作平台，我希望做到的效果如下：
1. 用户使用浏览器访问系统，系统界面类似于常见的编写代码的 ai ide（如 trae、cursor 等）：顶部导航条、左侧目录区、中间是文件内容、右侧是对话区、底部状态栏
2. 一本小说就可以类比一个项目，最顶层目录的分类是：创作区 和 归档区，
  a. 创作区包括创作过程的所有内容，编写或修改哪一章节，就将当前的主工作区切换到该章节对应的目录
  b. 归档区是已完成的章节，通常不允许修改
3. 通常一个小说项目需要维护：总纲、时间线、角色、世界背景、世界架构、伏笔、灵感、写作风格等，并且可以随小说剧情发展修改增补内容
4. 小说以 卷、章节 的形式维护，卷与章节就是现今网络小说的常用方式
5. 每个章节的工作区还需要维护：章节大纲、主要场景 等，用户在右侧的对话区与 ai agent 对话，可以生成/修改 当前章节的大纲、场景、正文等
6. 可以与 agent 讨论全书总纲，也可以进行发散式地讨论，输出灵感、伏笔等
7. 系统可使用 git 维护书本的创作过程，并为用户提供简单的版本查看、比较的功能，同样是与 ide 对比代码的模式一样
8. 系统配置页面可以修改系统参数


你需要认真思考，可以参考行业内的成熟方案完善需求，然后输出需求文档，文档中不要包含代码，现阶段只讨论需求。必要的话可以输出 html 原型。
