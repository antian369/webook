# AI 小说创作平台需求文档

## 1. 项目概述

### 1.1 产品定位
一个基于 AI Agent 的专业小说创作平台，采用类 IDE 的界面设计，为网文作者提供从灵感收集、世界观构建、角色设计、大纲规划到章节创作的完整工作流支持。

### 1.2 目标用户
- 网络小说作者（主要）
- 传统小说创作者
- 剧本/游戏文案创作者
- 写作爱好者

### 1.3 核心设计理念
- **项目化思维**：将小说类比软件项目，采用 IDE 式的项目管理方式
- **AI 协作**：Agent 不仅是生成工具，更是创作伙伴，支持多轮对话和上下文理解
- **版本控制**：类似 Git 的创作历史管理，支持回溯、比较和分支
- **结构化创作**：强制化的创作元素管理（角色、世界观、时间线等）

## 2. 功能需求

### 2.1 工作区架构

#### 2.1.1 小说项目结构
```
小说项目（类比代码仓库）
├── 📁 创作区/                # 当前活跃的创作内容
│   ├── 📁 总纲/              # 全书总纲文档
│   ├── 📁 时间线/            # 故事时间线
│   ├── 📁 角色/              # 角色资料库
│   ├── 📁 世界背景/          # 世界设定
│   ├── 📁 世界架构/          # 力量体系、势力分布等
│   ├── 📁 伏笔/              # 伏笔管理
│   ├── 📁 灵感/              # 灵感笔记
│   ├── 📁 写作风格/          # 风格指南
│   └── 📁 正文/              # 小说正文（按卷组织）
│       ├── 📁 第一卷：卷名/
│       │   └── 📁 第一章：章名/        # 章节作为目录
│       │       ├── 📄 章节大纲.md      # 本章概要
│       │       ├── 📄 章节背景.md      # 本章背景设定
│       │       ├── 📁 主要场景/        # 场景列表
│       │       │   ├── 📄 场景1.md
│       │       │   └── 📄 场景2.md
│       │       └── 📄 正文.md          # 章节正文
│       ├── 📁 第二章：章名/
│       │   ├── 📄 章节大纲.md
│       │   ├── 📄 章节背景.md
│       │   ├── 📁 主要场景/
│       │   └── 📄 正文.md
│       └── 📁 第三卷：卷名/
│           └── 📁 第一章：章名/
│               ├── 📄 章节大纲.md
│               ├── 📄 章节背景.md
│               ├── 📁 主要场景/
│               └── 📄 正文.md
└── 📁 归档区/                # 已归档的章节正文
    └── 📁 [卷名]/            # 按卷归档
        └── 📄 第一章：章名.md   # 归档的章节正文（只读）
```

#### 2.1.2 创作区规则
- **可编辑性**：创作区内所有文档均可编辑
- **实时同步**：编辑内容实时保存
- **工作区切换**：打开某一章节时，工作区自动切换到该章节目录
- **上下文继承**：AI Agent 自动获取当前工作区的上下文（章节大纲、场景、角色信息等）
- **章节目录结构**：每个章节作为独立目录，包含大纲、背景、场景子目录和正文文件

#### 2.1.3 归档区规则
- **只读模式**：归档区内容默认只读
- **归档条件**：用户完成一个章节后，**手动执行归档操作**，仅对该章节的**正文文件**进行归档
- **归档操作**：
  - 在章节目录上右键选择"归档章节"
  - 系统将该章节的 `正文.md` 复制到归档区对应位置
  - 原创作区的正文文件保留，可继续编辑
  - 归档的正文标记为"已归档"状态
- **归档管理**：
  - 在归档区可查看所有已归档的章节正文
  - 支持对比创作区和归档区的正文差异
  - 可导出归档的章节为独立文档
- **重新激活**：如需修改已归档内容，在归档区右键选择"重新激活"，将归档版本合并回创作区

### 2.2 核心功能模块

#### 2.2.1 全局管理文档

##### 2.2.1.1 总纲
- **功能**：维护全书核心设定
- **内容项**：
  - 书名、作者、类型、标签
  - 一句话简介
  - 核心卖点/亮点
  - 故事主线（起承转合）
  - 主题与核心思想
  - 目标字数
  - 预计卷数
- **AI 交互**：可与 Agent 讨论总纲合理性、优化方向

##### 2.2.1.2 时间线
- **功能**：维护故事的时间脉络
- **内容项**：
  - 支持多条时间线（主线、支线、角色个人线）
  - 时间点/时间段标记
  - 每个时间点的事件描述
  - 关联角色、地点
  - 时间线可视化（类甘特图）
- **AI 交互**：检查时间逻辑一致性、发现时间冲突

##### 2.2.1.3 角色管理
- **功能**：维护所有角色资料
- **内容项**：
  - 基础信息：姓名、年龄、性别、外貌、身份
  - 性格特征：MBTI、性格标签、口头禅
  - 背景故事：出身、经历、动机
  - 能力/技能：力量体系相关
  - 人际关系：关系图谱可视化
  - 成长弧线：角色转变轨迹
  - 备注/备注标签
- **AI 交互**：生成角色设定、完善角色背景、检查角色一致性

##### 2.2.1.4 世界背景
- **功能**：维护世界观设定
- **内容项**：
  - 地理：地图、国家、城市、地点
  - 历史：大事记、年代划分
  - 文化：习俗、宗教、语言、艺术
  - 种族：各类种族/物种设定
  - 科技/魔法：力量体系详细规则
- **AI 交互**：扩展世界细节、生成地点描述、检查设定一致性

##### 2.2.1.5 世界架构
- **功能**：维护力量体系、势力分布等结构性设定
- **内容项**：
  - 力量体系：等级划分、能力描述
  - 势力分布：势力图谱、关系网络
  - 规则系统：世界运行规则
  - 物品/道具：重要道具设定
- **AI 交互**：设计平衡的力量体系、生成势力冲突

##### 2.2.1.6 伏笔管理
- **功能**：维护全书伏笔
- **内容项**：
  - 伏笔列表：描述、埋设章节、预计回收章节
  - 状态跟踪：已埋设/已回收/废弃
  - 回收提醒：到达预计回收点时提醒
  - 关联关系：伏笔之间的关联
- **AI 交互**：发现潜在伏笔、检查伏笔回收情况、生成回收方案

##### 2.2.1.7 灵感笔记
- **功能**：收集创作灵感
- **内容项**：
  - 灵感条目：标题、内容、标签
  - 来源标记：想法来源
  - 状态：待处理/已采纳/已丢弃
  - 关联：可关联到具体章节或角色
- **AI 交互**：对灵感进行发散讨论、将灵感整合进故事

##### 2.2.1.8 写作风格
- **功能**：定义和维护写作风格
- **内容项**：
  - 语言风格：古风/现代/西幻等
  - 叙事视角：第一人称/第三人称有限/全知
  - 叙述节奏：快节奏/慢热
  - 对白风格：简洁/华丽
  - 示例片段：风格示例
  - 禁用词/禁用表达
- **AI 交互**：分析现有文本风格、根据风格生成内容

#### 2.2.2 章节工作区

##### 2.2.2.1 章节目录结构（文件夹形式）
每个章节作为一个独立的目录（文件夹）存在，目录中包含：

- **章节大纲.md**：本章节的概要文档
  - 本章目标：本章在全书中的作用
  - 关键事件：本章必须发生的核心事件
  - 情绪基调：整体氛围和节奏
  - 出场角色：预计出场的角色清单
  - 地点：主要场景地点
  - 伏笔埋设：本章要埋设的伏笔
  - 与前后章的衔接点

- **章节背景.md**：本章特有的背景设定
  - 时间背景：故事发生的时间节点
  - 环境描述：天气、场景氛围等
  - 特殊设定：本章涉及的特殊规则或设定
  - 前置条件：阅读本章前需要知道的信息

- **主要场景/**：场景子目录
  - 每个场景为独立的 `.md` 文件
  - 场景文件命名：`场景1.md`、`场景2.md` 或 `场景_逃离追杀.md`
  - 场景内容作为正文的参考或草稿

- **正文.md**：章节的最终正文内容
  - 纯文本编辑器（类似 VS Code）
  - 支持 Markdown 语法
  - 字数统计和进度显示
  - 可引用场景文件的内容

##### 2.2.2.2 场景管理
- **场景文档**：
  - 场景目标：本场景的作用
  - 场景概要：简要描述
  - 出场角色：角色列表
  - 地点：场景发生地
  - 时间：故事内时间
  - 情绪基调：场景氛围
  - 内容：场景正文（可作为正文参考）
- **场景操作**：增删改查、调整顺序、合并拆分
- **场景与正文的关系**：
  - 场景文件是草稿/参考，正文是最终成果
  - 可将场景内容一键合并到正文
  - 归档时只归档正文，不归档场景草稿

##### 2.2.2.3 正文编辑器
- **纯文本编辑器**：类似 VS Code 的代码编辑器体验
  - 支持语法高亮（Markdown）
  - 行号显示
  - 代码折叠（按标题、场景等）
  - 多光标编辑
  - 查找替换（支持正则）
- **字数统计**：实时字数、目标字数进度
- **写作模式**：专注模式（全屏）、普通模式
- **自动保存**：定时自动保存到本地存储
- **智能提示**：
  - 角色名自动补全
  - 地点名自动补全
  - 前文提及的关键词补全
- **章节预览**：分屏预览 Markdown 渲染效果

### 2.3 AI Agent 功能

#### 2.3.1 对话模式

##### 2.3.1.1 上下文感知
- **自动上下文**：Agent 自动获取当前工作区内容
  - 打开总纲时：了解全书背景
  - 打开章节时：获取章节大纲、前文内容
  - 打开角色时：了解角色设定
- **上下文控制**：用户可手动添加/移除上下文引用

##### 2.3.1.2 对话类型
1. **创作型对话**
   - 生成大纲：根据总纲生成卷/章大纲
   - 生成场景：根据大纲生成场景内容
   - 续写：根据前文续写正文
   - 改写：改进现有文本
   - 扩写：扩展简短内容

2. **讨论型对话**
   - 头脑风暴：发散讨论创意
   - 剧情讨论：讨论剧情走向
   - 角色讨论：讨论角色发展
   - 设定讨论：完善世界设定

3. **检查型对话**
   - 一致性检查：检查设定前后一致性
   - 逻辑检查：检查剧情逻辑
   - 节奏检查：检查叙事节奏
   - 风格检查：检查是否符合设定风格

#### 2.3.2 快捷操作
- **选中操作**：选中文本后弹出快捷菜单
  - 改写：改进表达
  - 扩写：扩展内容
  - 简化：精简表达
  - 润色：改善文笔
  - 描述：添加感官细节
  - **添加到对话引用**：将选中文本作为引用添加到右侧对话区（快捷键 Ctrl+I）
- **一键生成**：
  - 根据大纲生成章节
  - 根据场景生成正文
  - 生成角色卡
  - 生成地点描述

#### 2.3.3 工作区文本引用功能（Cursor Ctrl+I 模式）
- **功能描述**：用户在中间编辑区选中文本后，可快速将其作为引用插入到右侧 AI 对话区
- **触发方式**：
  - 快捷键：`Ctrl+I`（类似 Cursor 的 Composer 功能）
  - 浮动工具栏：选中文本后点击"引用到对话"按钮
  - 右键菜单：右键选中文本选择"添加到 AI 对话"
- **引用格式**：
  ```
  > 📄 来自「第一章：觉醒之日」正文
  > 林凡深吸一口气，握紧了拳头。他知道，这是他唯一的机会...
  
  [用户输入问题或指令]
  ```
- **交互流程**：
  1. 用户在编辑器中选中文本
  2. 按 `Ctrl+I` 或点击引用按钮
  3. 系统自动在右侧对话区添加引用块
  4. 输入框自动聚焦，等待用户输入问题
  5. 发送后，AI 基于引用内容回复
- **多段引用支持**：
  - 支持多次 `Ctrl+I` 添加多个引用
  - 引用按添加顺序排列
  - 可点击引用块上的 × 移除某个引用
- **引用来源标识**：
  - 显示文件名和文档类型图标
  - 显示引用文本的行号范围
  - 点击引用可跳转回原文位置

#### 2.3.4 Agent 设置
- **模型选择**：选择使用的 AI 模型
- **温度调节**：控制生成创意度
- **系统提示词**：自定义 Agent 角色设定
- **记忆管理**：管理 Agent 的长期记忆

### 2.4 版本控制

#### 2.4.1 Git 集成
- **自动提交**：设定时间间隔自动提交（如每 30 分钟）
- **手动提交**：用户手动创建提交点
- **提交信息**：AI 辅助生成提交说明
- **分支管理**：
  - 主线：主创作线
  - 实验分支：尝试性修改
  - 归档分支：已归档内容的冻结版本

#### 2.4.2 版本历史
- **提交列表**：按时间倒序显示提交历史
- **时间线视图**：可视化提交图谱（类似 GitKraken）
- **文件历史**：查看单个文件的修改历史

#### 2.4.3 比较功能
- **版本比较**：选择两个提交点比较差异
- **Diff 视图**：
  - 行级差异高亮（增删改）
  - 字符级差异（精确到字）
  - 合并冲突解决界面
- **文件比较**：支持章节、大纲等文档的比较

#### 2.4.4 回滚与恢复
- **单文件回滚**：将单个文件恢复到历史版本
- **整章回滚**：将整个章节恢复到历史状态
- **软恢复**：查看历史版本内容但不覆盖当前
- **硬恢复**：完全回退到历史版本

### 3.2 区域详细说明

#### 3.2.1 顶部导航栏
- **高度**：48px
- **背景**：深色（暗色主题）或浅色

## 4. 交互需求

### 4.1 文件操作
- **新建**：右键菜单或快捷键 Ctrl+N
- **重命名**：F2 或右键菜单
- **删除**：Delete 键（提示确认）
- **移动**：拖拽或剪切粘贴
- **复制**：Ctrl+C / Ctrl+V

### 4.2 AI 交互
- **发送消息**：Enter（换行 Shift+Enter）
  - 输入框显示提示："按 Enter 发送，Shift+Enter 换行"
- **引用上下文**：拖拽文档到对话区或 @ 符号引用
- **快捷操作**：
  - 选中文字后显示浮动工具栏
  - 预设快捷按钮（生成、改写、扩写）
- **中断生成**：生成过程中显示停止按钮

### 4.3 编辑器交互
- **自动保存**：输入停止 3 秒后自动保存
- **手动保存**：Ctrl+S
- **撤销/重做**：Ctrl+Z / Ctrl+Y（或 Ctrl+Shift+Z）
- **查找/替换**：Ctrl+F / Ctrl+H
- **分屏编辑**：支持左右分屏对比编辑

### 4.4 版本控制交互
- **提交**：Ctrl+Enter（快速提交）或详细提交对话框
- **查看历史**：点击文件右键"查看历史"
- **比较版本**：在历史列表中选择两个版本
- **回滚**：在历史详情中点击"恢复此版本"

## 7. 用户故事

### 7.1 核心用户场景

#### 场景 1：开始新小说
1. 用户创建新项目，填写基本信息（书名、类型、目标字数）
2. 系统生成标准目录结构
3. 用户在总纲中编写全书概要
4. 用户与 AI 讨论总纲，获取建议
5. 用户创建角色和世界观设定
6. 用户创建第一卷大纲

#### 场景 2：日常创作（基于章节目录）
1. 用户在目录树中点击章节目录展开
2. 用户看到章节下的文件结构：大纲、背景、场景、正文
3. 用户点击"正文.md"在编辑器中打开
4. 用户在编辑器中编写内容
5. 用户选中某段文字，按 `Ctrl+I` 添加到 AI 对话引用
6. AI 基于引用的上下文给出改写建议
7. 用户保存，系统自动创建 Git 提交点

#### 场景 3：使用 Ctrl+I 引用功能
1. 用户在编辑器中编写正文时遇到卡文
2. 用户选中前文的一段描写
3. 用户按 `Ctrl+I` 快捷键
4. 系统将该段文字作为引用块添加到右侧对话区
5. 用户输入问题："帮我把这段写得更紧张一些"
6. AI 分析引用内容，给出改写建议
7. 用户点击"应用"将建议插入到编辑器

#### 场景 4：归档章节
1. 用户完成第一章的全部写作
2. 用户在目录树中右键第一章目录
3. 选择"归档章节"选项
4. 系统提示："确定要归档第一章吗？这将把正文.md 复制到归档区"
5. 用户确认后，正文被复制到归档区并标记为只读
6. 创作区的正文文件仍保留，可继续编辑
7. 归档区的文件显示 🔒 图标，表示已归档

#### 场景 5：修改已归档内容
1. 用户发现已归档章节中的逻辑错误
2. 用户在归档区右键该章节文件
3. 选择"重新激活"选项
4. 系统询问是"创建新版本"还是"覆盖当前"
5. 用户选择"创建新版本"
6. 归档的正文被复制回创作区，并标记为新版本
7. 用户修改后，可以再次归档或保留在创作区

#### 场景 6：回顾创作历程
1. 用户打开版本控制界面
2. 用户查看提交历史时间线
3. 用户选择两个时间点比较差异
4. 用户发现某次修改有问题
5. 用户回滚到之前的版本
6. 用户导出特定版本的 PDF

## 9. 风险与限制

### 9.1 技术风险
- **AI 依赖性**：需要用户自备 AI API Key
- **大文件性能**：超大文本文件的编辑性能
- **浏览器限制**：本地文件系统访问限制

### 9.2 产品风险
- **学习曲线**：IDE 界面可能对非技术用户不友好
- **AI 幻觉**：AI 生成内容可能不符合设定
- **过度依赖**：作者可能过度依赖 AI 而失去创作能力

### 9.3 缓解方案
- 提供详细的使用教程和模板
- 提供 AI 一致性检查工具
- 强调 AI 作为辅助而非替代
- 提供多种界面模式（简化/专业）

## 11. 附录

### 11.1 术语表
- **卷**：小说的第一级结构，通常包含多个章节
- **章**：小说的基本单位，通常 2000-5000 字
- **章节目录**：每个章节作为独立目录，包含大纲、背景、场景、正文等文件
- **章节大纲**：单个章节的概要文档，描述本章目标和关键事件
- **章节背景**：本章特有的背景设定和前置条件
- **场景**：章内的子单位，一个完整的小事件，作为草稿/参考文件
- **正文**：章节的最终正文内容，可归档的正式版本
- **Agent**：AI 助手，可以与用户对话并生成内容
- **上下文**：AI 可以看到的当前相关信息
- **工作区**：当前活跃的编辑区域
- **归档**：用户手动将章节正文标记为只读，保存到归档区
- **Ctrl+I 引用**：将编辑区选中的文本快速添加到 AI 对话区的功能
