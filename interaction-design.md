# AI 小说创作平台 - 交互流程与界面规范

## 1. 核心交互流程

### 1.1 新建小说项目流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1.创建项目  │ ──▶ │  2.填写信息  │ ──▶ │ 3.系统生成  │ ──▶ │  4.开始创作  │
│   (入口)    │     │  (配置页)   │     │  (目录结构) │     │  (工作台)   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
     │                   │                   │                   │
     ▼                   ▼                   ▼                   ▼
• 点击"新建项目"      • 书名/作者       • 创作区文件夹     • 自动打开总纲
• 选择模板           • 类型/标签        • 归档区文件夹     • AI助手问候
• 导入现有文档       • 目标字数        • 示例文档         • 引导使用
```

### 1.2 日常创作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1.选择章节  │ ──▶ │  2.AI辅助    │ ──▶ │  3.编辑内容  │ ──▶ │  4.保存提交  │
│  (左侧目录) │     │  (右侧面板)  │     │  (中间编辑)  │     │  (自动/手动) │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
     │                   │                   │                   │
     ▼                   ▼                   ▼                   ▼
• 点击卷/章节展开    • 查看大纲建议    • 编写/改写       • 自动保存
• 工作区自动切换    • 生成场景内容    • 扩写内容        • Git自动提交
• 上下文自动加载    • 讨论剧情发展    • 润色语言        • 手动创建版本
```

### 1.3 AI 协作交互流程

```
用户操作                      系统响应
─────────────────────────────────────────
输入指令 ───────────────────▶ AI处理中
                              (显示typing)
                              
                              ▼
                              
◀────────────────────────── 生成回复
显示在对话区
                              
点击"应用" ─────────────────▶ 插入到编辑器
或拖拽到编辑区

### 1.5 章节工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1.创建章节  │ ──▶ │  2.编写文档  │ ──▶ │  3.完成章节  │
│  (目录结构) │     │  (多文件编辑)│     │  (手动归档) │
└─────────────┘     └─────────────┘     └─────────────┘
     │                   │                   │
     ▼                   ▼                   ▼
• 新建章节目录      • 编写章节大纲      • 右键章节目录

  - 主要场景/       • 编写正文内容      • 保留创作区编辑
  - 正文.md
```

## 2. 界面状态说明

### 2.1 文件状态标识

| 图标 | 状态 | 说明 |
|------|------|------|
| 🟢 | 已保存 | 文档已保存，无未提交更改 |
| 🟡 | 未保存 | 有未保存的修改 |
| 🔵 | 未提交 | 已保存但未提交到 Git |
| ⚪ | 只读 | 归档区文件，不可编辑 |
| 🔒 | 锁定 | 被其他协作者锁定（未来功能） |

### 2.2 AI 助手状态

| 状态 | 显示 | 说明 |
|------|------|------|
| 在线 | 🟢 ● 在线 | AI服务正常，可响应请求 |
| 思考中 | ⚪ ◐ 思考中... | 正在生成回复 |
| 繁忙 | 🟡 ● 繁忙 | 队列中有多个请求 |
| 离线 | 🔴 ● 离线 | 连接失败，请检查设置 |
| 配置错误 | ⚠ 配置错误 | API Key 或模型配置有误 |

### 2.3 上下文引用状态

```

## 5. 提示词模板体系

### 5.1 系统预设提示词

#### 5.1.1 小说创作助手（默认）

```
你是一位专业的小说创作助手，擅长玄幻修仙题材。你的职责包括：

1. **内容创作**：根据大纲生成高质量章节内容，注意节奏把控和细节描写
2. **剧情讨论**：与作者讨论剧情走向，提供合理化建议
3. **设定检查**：检查设定的一致性，指出可能的逻辑漏洞
4. **角色塑造**：帮助完善角色设定，保持角色行为逻辑一致
5. **文笔润色**：改善文笔，增加描写，调整语气

**创作原则**：
- 保持网文的快节奏叙事风格
- 注重爽点的营造和铺垫
- 遵循"黄金三章"原则设计开头
- 注意伏笔的埋设和回收

**响应格式**：
- 直接给出可用的文本内容
- 对建议类问题，给出清晰的选项和理由
- 涉及修改时，说明修改理由
```

#### 5.1.2 剧情分析师

```
你是一位资深的剧情分析师，专门分析小说剧情的合理性和吸引力。

**分析维度**：
1. 剧情逻辑是否自洽
2. 节奏是否合理（张弛有度）
3. 冲突设置是否到位
4. 伏笔安排是否合理
5. 高潮是否足够精彩

**输出格式**：
- 总体评价（1-10分）
- 优点分析
- 改进建议（具体到段落）
- 可添加的细节建议
```

## 6. 数据流设计

### 6.1 自动保存机制

```
用户输入
    │
    ▼
防抖处理 (停止输入3秒后)
    │
    ▼
本地存储 (LocalStorage)
    │
    ├─────▶ 成功：显示"已保存"
    │
    └─────▶ 失败：显示"保存失败"，保留内存版本
    │
    ▼
文件系统写入 (后台)
    │
    ├─────▶ 成功：更新最后保存时间
    │
    └─────▶ 失败：标记为未保存状态
```

### 6.2 Git 自动提交机制

```
定时器触发 (默认30分钟)
    │
    ▼
检查是否有未提交的更改
    │
    ├─────▶ 无更改：跳过本次提交
    │
    └─────▶ 有更改：
              │
              ▼
        生成提交信息 (AI辅助)
              │
              ▼
        执行 git add
              │
              ▼
        执行 git commit
              │
              ├─────▶ 成功：显示提交通知
              │
              └─────▶ 失败：显示错误，保留到下次
```

### 6.3 AI 请求流程

```
用户发送消息
    │
    ▼
收集上下文
    ├── 当前文档内容
    ├── 引用的其他文档
    ├── 系统提示词
    └── 历史对话（最近5轮）
    │
    ▼
构造请求体
    │
    ▼
发送到 AI API
    │
    ├─────▶ 流式响应：逐字显示
    │
    └─────▶ 完整响应：一次性显示
    │
    ▼
渲染回复 (Markdown解析)
    │
    ▼
添加到对话历史
    │
    ▼
用户可选操作：
    ├── 复制内容
    ├── 应用到编辑器
    ├── 重新生成
    └── 继续对话
```

## 7. 错误处理与反馈

### 7.1 错误提示设计

| 场景 | 提示方式 | 用户操作 |
|------|----------|----------|
| 保存失败 | 底部状态栏红色闪烁 + Toast通知 | 重试/稍后重试 |
| AI请求失败 | 对话区错误提示 | 重试/检查设置 |
| Git提交失败 | 状态栏Git图标变红 | 查看详情/手动提交 |
| 文件冲突 | 弹窗提示 + Diff对比 | 选择保留版本 |
| 网络断开 | 全局提示条 | 等待恢复/离线模式 |

### 7.2 成功反馈设计

| 场景 | 反馈方式 | 持续时间 |
|------|----------|----------|
| 保存成功 | 状态栏更新 | 永久 |
| 提交成功 | Toast通知 "已创建版本 a1b2c3d" | 3秒 |
| AI生成完成 | 轻微震动（移动端）/声音提示（可选） | 即时 |
| 导出成功 | 下载开始 + Toast通知 | 3秒 |

## 10. 性能优化策略

### 10.1 大文件处理

| 文件大小 | 处理策略 |
|----------|----------|
| <100KB | 完整加载，实时编辑 |
| 100KB-1MB | 虚拟滚动，按需渲染 |
| >1MB | 分页加载，章节跳转 |

### 10.2 渲染优化

- **虚拟列表**：长目录树使用虚拟滚动
- **防抖节流**：搜索、自动保存使用防抖
- **懒加载**：图片、历史记录按需加载
- **Web Worker**：复杂计算移到后台线程
- **缓存策略**：文件内容本地缓存，哈希校验

### 10.3 AI 优化

- **请求合并**：短时间内多次请求合并为一次
- **流式输出**：AI回复流式显示，减少等待感
- **本地缓存**：常用上下文本地预加载
- **增量更新**：只发送变更内容，减少token消耗
